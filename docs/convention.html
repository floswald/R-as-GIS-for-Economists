0.5 Conventions of the book and some notes

Here are some notes of the conventions of this book and notes for \(R\) beginners and those who are not used to reading rmarkdown-generated html documents.

0.5.1 Texts in gray boxes

They are one of the followings:

  • objects defined on R during demonstrations
  • R functions
  • R packages

When it is a function, I always put parentheses at the end like this: st_read().9 Sometimes, I combine a package and function in one like this: sf::st_read(). This means it is a function called st_read() from the sf package.

0.5.2 Colored Boxes

Codes are in blue boxes, and outcomes are in red boxes.

Codes:

runif(5)

Outcomes:

## [1] 0.93791333 0.02976169 0.01848662 0.05063421 0.02579477

0.5.3 Parentheses around codes

Sometimes you will see codes enclosed with parenthesis like this:

(
  a <- runif(5)
)
## [1] 0.3770006 0.3385762 0.3054660 0.1087869 0.5523285

The parentheses prints what’s inside of a newly created object (here a) without explicitly evaluating the object. So, basically I am signaling that we will be looking inside of the object that was just created.

This one prints nothing.

a <- runif(5)

So, don’t get confused with the parentheses around some codes. They are basically just for my convenience.

  1. This is a function that draws values randomly from the uniform distribution.↩
